# BỘ TÀI LIỆU HỆ THỐNG SOCIAL BLOG

Thư mục `docs` đã được chuẩn hóa lại theo hướng tài liệu kỹ thuật dạng báo cáo.

## Danh mục tài liệu

1. `01-system-design-overview.md`  
   Tổng quan thiết kế hệ thống, mục tiêu, phạm vi chức năng, công nghệ và nguyên tắc thiết kế.

2. `02-system-components.md`  
   Mô tả chi tiết các thành phần trong hệ thống: frontend, backend, realtime, cơ sở dữ liệu và lưu trữ ảnh.

3. `03-data-and-api-design.md`  
   Trình bày mô hình dữ liệu nghiệp vụ, các nhóm API hiện có, đánh giá hiện trạng và hướng cải tiến.

4. `04-scalability-and-high-availability.md`  
   Phân tích kiến trúc chịu tải, phương án scale ngang, các điểm nghẽn hiện tại và lộ trình nâng cấp.

## Nguyên tắc biên soạn

- Văn phong báo cáo kỹ thuật
- Nội dung bám sát mã nguồn hiện tại
- Có bổ sung định hướng kiến trúc mục tiêu cho khả năng mở rộng và chịu tải
- Loại bỏ các tài liệu giai đoạn nhỏ lẻ, phân tán và trùng lặp
